# 삼성 라이온즈 팬페이지 데모

삼성 라이온즈 팬페이지 기획안을 바탕으로 제작한 정적 Next.js 데모 애플리케이션입니다. 메인, 선수 정보, 경기 기록, 커뮤니티, 관리자 콘솔 페이지를 구성해 향후 실제 개발 시 UX 흐름과 콘텐츠 배치를 미리 확인할 수 있습니다.

## 주요 페이지

| 경로 | 설명 |
| --- | --- |
| `/` | 최근 경기 결과, 다음 경기 일정, 팬 인기투표, 뉴스 카드 등을 담은 메인 대시보드 |
| `/players` | 포지션별 선수 카드, 최근 기록, 하이라이트 배치 예시 |
| `/records` | 일정/결과 캘린더 카드, 박스 스코어, 팀 성적 추이 시각화 가이드 |
| `/community` | 자유게시판, Q&A, 응원존 구성과 인터랙션 아이디어 |
| `/admin` | 역할 기반 권한, 관리자 업무, 인프라/보안 체크리스트 |

## 기술 스택

- **Next.js 14** – App Router 기반의 정적 페이지 구성
- **React 18** – UI 컴포넌트 설계
- **Tailwind CSS 3** – 빠른 스타일링과 반응형 레이아웃
- (선택) Recharts / Chart.js, Firebase, Supabase 등은 실제 서비스 연동 시 도입

## 로컬 실행 방법

1. 의존성 설치

   ```bash
   npm install
   ```

2. 개발 서버 실행

   ```bash
   npm run dev
   ```

3. 브라우저에서 `http://localhost:3000` 접속 후 각 메뉴 탐색

> **참고:** 본 저장소에는 백엔드나 실시간 기능이 포함되어 있지 않으며, 모든 데이터는 `data/` 디렉터리의 샘플 JSON으로 구성되어 있습니다.

## 프로젝트 구조

```
app/
  layout.js          # 공통 레이아웃, 네비게이션/푸터 포함
  page.js            # 메인 대시보드
  players/page.js    # 선수 정보 페이지
  records/page.js    # 경기 기록 페이지
  community/page.js  # 팬 커뮤니티 페이지
  admin/page.js      # 관리자 콘솔 페이지
components/          # 공통 UI 컴포넌트
data/                # 샘플 데이터 모음
```

## 향후 확장 아이디어

- KBO 또는 구단 API 연동으로 실시간 경기/선수 데이터 제공
- Firebase/Supabase 인증 및 실시간 응원 피드 구축
- Recharts/Chart.js를 이용한 성적 추이 그래프 구현
- 이미지 업로드 및 콘텐츠 필터링 기능, RBAC 기반 관리자 워크플로 구축
